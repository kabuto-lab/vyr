Анализ драматического падения FPS на середине боя

Краткое описание проблемы

Во время боя наблюдается драматическое падение FPS примерно на середине симуляции. Это может быть связано с увеличением количества вычислений, конкуренцией за ресурсы или другими факторами, характерными для середины игры.

1. Хронология боя

Ранняя фаза (ходы 1-50)

  • Мало занятых клеток (4 начальных колонии)
  • Мало взаимодействий между вирусами
  • Низкая плотность населения
  • Минимальные вычисления в фазе боя

Ранняя фаза (ходы 1-50)

  • Стабильный FPS
  • Минимальная нагрузка на CPU
  • Мало визуальных эффектов

Средняя фаза (ходы 50-150)

  • Увеличение плотности населения
  • Больше взаимодействий между вирусами
  • Конкуренция за территорию усиливается
  • Больше клеток участвует в фазе боя

Средняя фаза (ходы 50-150)

  • Начало падения FPS
  • Увеличение нагрузки на CPU
  • Больше визуальных эффектов (если не отключены)
  • Возможна утечка памяти

Поздняя фаза (ходы 150+)

  • Максимальная плотность населения
  • Интенсивные боевые действия
  • Максимальное количество взаимодействий
  • Постоянные захваты территории

Поздняя фаза (ходы 150+)

  • Стабилизация FPS на низком уровне
  • Максимальная нагрузка на CPU
  • Максимум визуальных эффектов (если не отключены)

2. Возможные причины драматического падения FPS на середине боя

Причина Описание Влияние на FPS Вероятность
Экспоненциальный рост взаимодействий Количество возможных взаимодействий между клетками растет экспоненциально с увеличением плотности населения -30% до -50% Высокая
Увеличение количества вычислений в фазе боя Больше клеток участвует в боевых действиях, больше проверок соседства -15% до -25% Высокая
Накопление визуальных эффектов Если визуальные эффекты не были полностью отключены, они могут накапливаться -10% до -20% Средняя
Утечка памяти Неосвобожденные объекты могут накапливаться и замедлять работу GC -10% до -30% Средняя
Ограничения браузера Ограничения на выполнение JavaScript или рендеринг в фоновых вкладках -5% до -15% Низкая
Конкуренция за ресурсы ОС Другие процессы могут начать использовать больше CPU/RAM в середине симуляции -5% до -10% Низкая

3. Подробный анализ фазы боя

Фаза боя (Phase 0) становится особенно затратной в середине игры из-за:

  • Увеличения плотности населения: больше клеток на поле означает больше возможных взаимодействий
  • Увеличения количества проверок соседства: каждая клетка проверяет своих 8 соседей, и чем больше клеток, тем больше проверок
  • Конкуренции за территорию: в середине игры начинается активная борьба за контроль над территорией
  • Повышенной активности вирусов: параметры вирусов начинают проявляться в полной мере

Формула для оценки сложности фазы боя: O(N * K), где:

  • N - количество занятых клеток
  • K - среднее количество соседей каждой клетки (обычно 8)

В ранней фазе N небольшое (~4-20 клеток), но в середине игры N может достигать 500-1000 клеток, что приводит к экспоненциальному росту вычислительной сложности.

4. Рекомендации по улучшению

  • Оптимизация фазы боя:
    • Использовать пространственные структуры данных (например, сетку или деревья) для быстрого поиска соседей
    • Ограничивать количество проверяемых клеток с помощью эвристик
    • Предварительно вычислять и кэшировать часто используемые значения (например, факторы атаки/защиты)
    • Оптимизировать алгоритмы выбора целей для атаки
  • Оптимизация визуальных эффектов:
    • Полностью отключить визуальные эффекты, как было сделано ранее
    • Если эффекты нужны, реализовать систему приоритетов и ограничения количества одновременных эффектов
    • Использовать более легковесные эффекты в середине и конце игры
  • Оптимизация памяти:
    • Регулярно очищать неиспользуемые объекты
    • Использовать object pooling для часто создаваемых/удаляемых объектов
    • Мониторить использование памяти и выявлять утечки
  • Адаптивная сложность:
    • Реализовать систему, которая адаптирует сложность вычислений в зависимости от текущего FPS
    • Снижать частоту обновлений при падении производительности

5. Мониторинг и диагностика

Для выявления точной причины падения FPS рекомендуется:

  • Использовать инструменты профилирования браузера (DevTools) для анализа производительности
  • Добавить метрики производительности в код для отслеживания FPS, количества объектов, использования памяти
  • Создать систему логирования, которая записывает состояние игры в моменты падения FPS
  • Анализировать количество активных клеток, взаимодействий и визуальных эффектов в разные фазы игры
