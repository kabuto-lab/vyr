In-App Purchase (IAP) Implementation Examples — VYRUS v2

1. Косметические улучшения

1.1. Темы оформления

  • Темы для вирусов: Хранить цветовые схемы в JSON или INI: { virus1: "#ff5252", virus2: "#4caf50", ... }
  • Анимированные эффекты: Использовать CSS keyframes или SVG SMIL для уникальных анимаций (например, пульсация, мерцание)
  • Формы клеток: Реализовать через SVG-паттерны или Canvas draw functions с переключением по типу: drawCell(type: 'circle' | 'square' | 'star')

1.2. Визуальные эффекты

  • Эффекты движения: Добавлять трейлы/следы при перемещении через временные canvas overlays
  • Специальные эффекты боя: Взрывы/искры через particle.js или собственный WebGL-шейдер
  • Эффекты захвата территории: Wave-анимация от центра новой клетки с использованием radial-gradient + transition

1.3. Интерфейс

  • Темы интерфейса: CSS переменные + localStorage для сохранения выбранной темы
  • Анимированные иконки: Lottie или inline SVG с SMIL-анимацией
  • Персонализированные фоны: Загружать фон из assets/backgrounds/[id].jpg и применять через document.body.style.backgroundImage

2. Функциональные улучшения

2.1. Улучшенные параметры

  • Премиум-слоты: Проверка подписки перед увеличением лимита очков: maxPoints = isPremium ? 20 : 16
  • Бонус к параметрам: Временный модификатор в state: { speed: baseSpeed + (hasBooster ? 2 : 0) }
  • Уникальные комбинации: Блокировка UI-элементов до покупки: virusPreset.locked = !userOwns('elite_preset')

2.2. Специальные способности

  • Особые вирусы: Отдельный класс VirusWithAbility extends Virus с методом activateSpecial()
  • Способности на карте: Кнопка активации в UI → вызов game.useAbility('heal_zone')
  • Ускорители: Таймер boostDuration в game state; обновление скорости симуляции через setInterval(..., baseDelay / (isBoosted ? 2 : 1))

2.3. Инструменты анализа

  • Расширенная статистика: Экспорт данных в CSV или отображение в модальном окне с графиками Chart.js
  • Анализ противников: API-запрос к /api/opponent-stats?id=... → отображение в таблице
