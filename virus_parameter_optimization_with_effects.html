Оптимизация параметров вирусов в VYRUS (с учетом визуальных эффектов)

Параметр 1: Агрессия (Aggression)

Влияние: определяет склонность вируса к атакам и захвату территории.

Механизм работы:

При значении >12 вирус может захватывать 2-3 соседние клетки за раз (взрывной рост).

ТИПА: При значении >12 происходит экспоненциальный рост вычислений из-за проверки нескольких клеток в одном направлении. Установка значения ≤12 значительно снижает нагрузку на CPU.

Влияние на FPS:

Высокое - при значениях >12 может вызывать до 3-кратного увеличения вычислений в фазе боя.

Параметр 2: Мутация (Mutation)

Влияние: влияет на способность вируса изменять свои характеристики и адаптироваться.

Механизм работы:

При значениях >8 создает визуальные эффекты (blob-эффекты) и может влиять на поведение вируса.

ТИПА: При значении >8 создается до 5 дополнительных визуальных элементов (blob-эффектов) на клетку, что значительно увеличивает нагрузку на рендеринг. Установка значения ≤8 уменьшает количество визуальных эффектов и повышает FPS.

Влияние на FPS:

Среднее - визуальные эффекты при высоких значениях могут снижать FPS на 5-10%.

Параметр 3: Скорость (Speed)

Влияние: определяет скорость перемещения и реакции вируса.

Механизм работы:

При значениях >8 может изменять визуальные эффекты.

ТИПА: Этот параметр в текущей реализации не имеет значительного влияния на вычисления, но при значениях >8 может создавать дополнительные визуальные эффекты. Установка значения ≤8 может немного снизить нагрузку на рендеринг.

Влияние на FPS:

Низкое - не влияет напрямую на вычислительную сложность.

Параметр 4: Защита (Defense)

Влияние: определяет способность вируса сопротивляться атакам.

Механизм работы:

Используется в вычислениях шанса защиты: (resistance/16 + defense/16) / 2. При значениях >8 может создавать визуальные эффекты.

ТИПА: Этот параметр не увеличивает вычислительную сложность, но при значениях >8 может создавать визуальные эффекты. Установка значения ≤8 может немного снизить нагрузку на рендеринг.

Влияние на FPS:

Низкое - не влияет на вычислительную сложность, только на визуальные эффекты.

Параметр 5: Воспроизводство (Reproduction)

Влияние: определяет способность вируса к размножению и расширению.

Механизм работы:

Влияет на шанс расширения: reproduction/16 используется в формуле (reproduction/16 + mobility/16 + infectivity/16) / 3.

ТИПА: Этот параметр влияет на шанс расширения, но не увеличивает вычислительную сложность. Однако при высоких значениях может увеличиваться плотность населения, что косвенно влияет на производительность. Оптимальное значение ≤10 позволяет сбалансировать эффективность и производительность.

Влияние на FPS:

Косвенное - через увеличение плотности населения, что увеличивает количество взаимодействий.

Параметр 6: Сопротивление (Resistance)

Влияние: определяет устойчивость вируса к внешним воздействиям.

Механизм работы:

Используется в вычислениях шанса защиты: (resistance/16 + defense/16) / 2. При значениях >8 может создавать визуальные эффекты.

ТИПА: Как и параметр защиты, не увеличивает вычислительную сложность, но при значениях >8 может создавать визуальные эффекты. Установка значения ≤8 может немного снизить нагрузку на рендеринг.

Влияние на FPS:

Низкое - не влияет на вычислительную сложность, только на визуальные эффекты.

Параметр 7: Скрытность (Stealth)

Влияние: определяет способность вируса избегать обнаружения и атаковать на расстоянии.

Механизм работы:

При значении >10 вирус может использовать скрытное расширение, проверяя клетки на расстоянии 2-3 шагов.

ТИПА: КРИТИЧЕСКИЙ ПАРАМЕТР! При значении >10 происходит значительное увеличение вычислений - проверка клеток на расстоянии 2-3 шагов требует O(D²) дополнительных вычислений, где D - расстояние. Установка значения ≤10 может повысить FPS на 15-25%!

Влияние на FPS:

Очень высокое - при значениях >10 может увеличивать вычислительную сложность в 5-10 раз.

ТИПА: В текущей реализации не имеет значительного влияния на вычисления, но при значениях >8 может создавать дополнительные визуальные эффекты. Установка значения ≤8 может немного снизить нагрузку на рендеринг.

ТИПА: При значениях >8 создает визуальные эффекты (например, белые точки в центре клетки), что увеличивает нагрузку на рендеринг. Установка значения ≤8 уменьшает количество визуальных эффектов и повышает FPS.

ТИПА: При значении >10 добавляется логика выбора направления, что требует дополнительных вычислений. При значениях >8 может создавать визуальные эффекты. Установка значения ≤10 может немного снизить вычислительную нагрузку.

Влияние на FPS:

Однако при высоких значениях может увеличиваться активность вируса, что косвенно влияет на производительность. При значениях >8 может создавать визуальные эффекты. Параметр 16: Стабильность (Stability)

Влияние: определяет стабильность вируса и его способность сохранять характеристики.

Механизм работы:

При значении >10 вирус использует упрощенную логику расширения и может создавать градиентные визуальные эффекты.

ТИПА: ИНТЕРЕСНЫЙ СЛУЧАЙ! При значении >10 вирус на самом деле УПРОЩАЕТ логику расширения, что может СНИЗИТЬ нагрузку на CPU. Однако при значениях >8 создает градиентные визуальные эффекты, что увеличивает нагрузку на рендеринг. Оптимальное значение 11-12 дает баланс между упрощенной логикой и минимальными визуальными эффектами.

Влияние на FPS:

Смешанное - упрощает вычисления, но создает визуальные эффекты.

Общие рекомендации по оптимизации

  • Критические параметры (ТИПА: КРИТИЧЕСКИЙ ПАРАМЕТР): Скрытность (>10) и Интеллект (>8) - их высокие значения значительно снижают FPS
  • Высокоопасные параметры: Агрессия (>12) - может вызвать экспоненциальный рост вычислений
  • Визуально нагруженные параметры: Мутация, Вирулентность, Летальность (>8) - создают дополнительные визуальные эффекты
  • Полезные параметры: Стабильность (>10) - упрощает логику вычислений, но создает визуальные эффекты
