Монетизация игры VYRUS (HTML5 / TypeScript / Web Workers)

Ваша игра — стратегический симулятор вирусов с сеткой, локальной и онлайн-многопользовательской поддержкой. Ниже — все возможные способы монетизации, адаптированные под ваш стек и тип игры.

1. Интеграция через игровые HTML5-платформы

Yandex Games

  • Модель: CPM (доход за показы рекламы), бонусы за вовлечённость
  • Реклама: Pre-roll, mid-roll, rewarded video (через YaGames API)
  • Требования: игра должна быть адаптирована под мобильные, поддерживать PWA или iframe, без внешних доменов (кроме разрешённых Yandex)
  • Интеграция:
    • Добавьте <script src="https://yandex.ru/games/sdk/v2">
    • Используйте YaGames.init().then(({ features }) => { features.AdvAPI.showFullscreenAdv(); })
    • Поддержка rewarded-видео позволяет давать игроку «бонусную мутацию» за просмотр
  • Плюсы: высокий трафик в СНГ, особенно Россия

VK Play (ранее My.Games)

  • Модель: CPM + микротранзакции (через VK Pay)
  • Реклама: rewarded video, interstitial
  • Требования: адаптация под мобильные, поддержка логина через VK ID
  • Интеграция:
    • Добавьте vkBridge и vk-bridge SDK
    • Используйте vkBridge.send("VKWebAppShowNativeAds", { ad_format: "reward" })
    • Для внутриигровых покупок — VK Pay (например, купить «особый штамм вируса»)

CrazyGames, Poki, GamePix

  • Модель: revenue share (обычно 30–50% от рекламных доходов)
  • Реклама: их собственные SDK (обычно без изменений в коде, но требуют whitelabel-релиз)
  • Подходит для: глобального трафика
  • Особенность: не требуют изменения UI, но могут накладывать свои баннеры

2. Прямая монетизация на вашем домене

Google AdSense / Ezoic / Mediavine (для веб-сайта)

  • Вставьте рекламные блоки на страницу игры (например, над/под canvas)
  • Требуется домен и трафик >10k просмотров/мес (AdSense — от 0, но с риском бана при низком качестве)
  • Не подходит для Vercel-домена — нужно своё доменное имя

Playwire / AdGem / AdColony (HTML5 In-Game Ads)

  • Специализируются на HTML5-играх
  • Предоставляют JS-SDK для rewarded video
  • Пример: игрок смотрит видео → получает +10 к «Мутации»

3. Внутриигровые покупки (если добавите онлайн-аккаунты)

Stripe / PayPal / Tinkoff (через ваш backend)

  • Продажа виртуальных предметов: уникальные вирусы, цвета, способности
  • Требует:
    • Backend (Node.js/Bun)
    • Аккаунт-систему (сохранение прогресса)
    • PCI-совместимую обработку платежей (лучше через Stripe Checkout)
  • Можно реализовать «Battle Pass» — как вы планировали в BattleServer

Telegram / VK Mini Apps

  • Разместите игру как мини-приложение
  • Монетизация через встроенные платёжные системы (Telegram Stars, VK Pay)
  • Высокий виральный коэффициент

4. Альтернативные методы

  • Donation / Boosty / Patreon: добавьте кнопку «Поддержать разработчика» (подходит для нишевой аудитории)
  • Лицензирование: продажа исходного кода или white-label версий другим студиям
  • Краудфандинг: запуск на Patreon с эксклюзивами (ранний доступ к новым вирусам)
Рекомендация для VYRUS:
Начните с Yandex Games + VK Play — ваша аудитория в СНГ, игра на русском/английском.
Добавьте rewarded video за:
  • Дополнительную мутацию перед боем
  • Реванш после поражения
  • Разблокировку редкого вируса
Это не нарушает баланс, но даёт доход.

5. Технические шаги для интеграции

  1. Создайте отдельную сборку без рекламы для GitHub (open-source версия)
  2. Создайте ads/ ветку или monetized/ сборку с интеграцией SDK
  3. Используйте Vite environment variables: import.meta.env.VITE_AD_PROVIDER
  4. Оберните вызовы рекламы в if (isMonetizedBuild)
  5. Не храните ключи API в клиентском коде — вызывайте через ваш Bun-бэкенд (если есть)
